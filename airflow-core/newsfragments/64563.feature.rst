Add ``@result`` decorator to mark a TaskFlow task as the DAG's result task. When ``@dag`` is used, returning a task's ``XComArg`` from the decorated function also designates it as the result automatically. The result task's return value is included in the ``/dags/{dag_id}/dagRuns/{dag_run_id}/wait`` API response by default when the ``result`` query parameter is not explicitly set.
