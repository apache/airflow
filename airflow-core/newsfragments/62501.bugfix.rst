Fix a race condition that could cause asset-triggered Dags to be skipped when asset events were produced concurrently. The scheduler could miss a newly produced asset event because there was a visibility gap between when the record was flushed and when the producing task's transaction committed. The asset event is now committed ahead of the producing task's remaining bookkeeping to close that gap; in rare cases this can surface as a duplicate or orphaned asset event if that task fails immediately after. Asset-triggered Dags now run reliably when their upstream assets are updated.
