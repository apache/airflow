The Airflow UI now uses the new index-friendly ``*_prefix_pattern`` REST API query parameters instead of the existing full-match ``*_pattern`` parameters on list endpoints. This is a behavioral change for search-as-you-type filters in the UI: matches are now prefix-based (``LIKE 'term%'`` via a range scan) instead of substring-based (``ILIKE '%term%'``), which means the database can use B-tree indexes and search stays fast on large deployments. The REST API itself keeps both forms: existing ``*_pattern`` parameters still behave exactly as before.
