Prime the ``dag_run`` relationship on newly-created mapped task instances during dynamic task mapping expansion, so the per-index ``TaskInstance.get_dagrun()`` calls made during dependency evaluation (e.g. ``NotPreviouslySkippedDep``) are cache hits instead of issuing a ``SELECT dag_run`` per mapped index. All mapped task instances in a run share one ``DagRun``, which is already in scope during expansion, so priming it removes this N+1 with no extra query.
