Package ``apache-airflow-providers-sandbox``
============================================

Run each Airflow task instance in an ephemeral cloud sandbox
(local / Daytona / E2B / Modal / islo) behind a pluggable provider layer.

The ``SandboxExecutor`` implements the public ``BaseExecutor`` interface
(AIP-51) and requires no core changes. Each queued task leases a fresh,
isolated sandbox, runs the task inside it, reports the exit state, and tears the
sandbox down — what ``KubernetesExecutor`` does for pods, generalised to
ephemeral cloud sandboxes.

Select it via ``[core] executor`` (alone or in a multi-executor list), choose a
backend with ``[sandbox] provider``, and route per-task overrides via
``executor_config["sandbox_override"]``. ``remote_logging`` is required: task
logs are produced inside the sandbox and shipped to remote storage by the
in-sandbox Task SDK supervisor.

Reference implementation and design notes:
https://github.com/zozo123/airflow-provider-sandbox
